# Hermes to Runtime Boundary

## Hermes owns
- user intent understanding
- skill routing
- session entry
- review lookup
- high-level memory summary
- cross-domain recommendation

## Runtime owns
- turn-by-turn conversation
- scenario control
- roleplay
- scoring
- diagnosis
- coaching
- compliance tagging
- progression update

## Principle
Hermes is thin.
Runtime is specialized.
